# 📊 RAG 성능 평가 결과 보고서 (v0.3.0)

> **문서 목적**: Job-Pocket RAG 시스템의 검색(Retrieval) 및 기술 매칭 성능 평가 결과를 기록한다.  
> **평가 일시**: 2026-04-22  
> **평가 대상**: HybridRetriever (FAISS + Tech Matcher)

---

## 1. 평가 개요

이번 평가는 약 4,000여 건의 대규모 데이터셋을 대상으로 HybridRetriever의 기술 정합성(TMR)을 정밀 검증하였습니다.

- **평가 모델**: Qwen3-Embedding-0.6B + FAISS
- **평가 규모**: 총 4,005개 샘플
- **핵심 지표**: **TMR (Tech Matching Rate)** - 사용자의 기술 스택과 검색된 자소서 간의 일치도

---

## 2. 주요 평가 결과

### 2.1 직무 및 순위별 TMR 통계

검색된 상위 3개 문서(Rank 1~3)에 대한 기술 스택 매칭률 결과입니다.

| 직무 (Position) | 1순위 (Rank 1) | 2순위 (Rank 2) | 3순위 (Rank 3) |
| :--- | :---: | :---: | :---: |
| **Frontend Engineer** | 83.27% | 82.19% | 81.94% |
| **Backend Engineer** | 78.70% | 77.71% | 77.28% |
| **AI Engineer** | 73.88% | 71.53% | 71.71% |
| **전체 평균 (TOTAL)** | **78.91%** | **77.50%** | **77.32%** |

### 2.2 결과 분석

1.  **Frontend 직군 우수**: Frontend 분야에서 약 83%의 높은 매칭률을 보였습니다. 이는 Frontend 관련 기술 키워드가 비교적 명확하고 임베딩 공간에서 잘 밀집되어 있음을 시사합니다.
2.  **AI 직군 개선 필요**: AI 직군의 경우 약 74%로 타 직군 대비 상대적으로 낮은 수치를 기록했습니다. AI 도메인의 기술 스택이 광범위하고 유사한 용어(예: ML/Deep Learning/AI)가 혼용되는 특성이 반영된 것으로 보입니다.
3.  **순위별 안정성**: 1순위부터 3순위까지의 점수 차이가 크지 않아(78.91% -> 77.32%), 상위 K개 문서 모두 일관된 품질의 기술적 맥락을 유지하고 있음을 확인했습니다.
4.  **Zero Matching Rate 0% 달성**: 모든 검색 결과에 대해 키워드가 한 건도 포함되지 않는 경우는 존재하지 않았습니다. 즉, 모든 결과 문서가 사용자의 기술 스택과 최소 하나 이상의 공통 분모를 가지고 있음을 검증했습니다.

---

## 3. 향후 개선 방향

- **동의어 사전 고도화**: AI 직군의 매칭률 향상을 위해 기술 스택 동의어 사전을 확장할 예정입니다.
- **Reranker 도입 검토**: 현재의 벡터 유사도 검색 결과를 기술 스택 기반으로 재정렬(Reranking)하여 1순위의 정합성을 90% 이상으로 끌어올리는 실험을 진행할 계획입니다.
- **임베딩 모델 튜닝**: 도메인 특화 용어에 더 민감하게 반응할 수 있도록 임베딩 모델의 Fine-tuning 또는 더 큰 모델(BGE-M3 등)로의 교체를 검토 중입니다.
